# Code of Conduct

Short and unsentimental. This is a single-maintainer open-source project.

## The rule

**Be hard on the code. Do not attack the person.**

That is the whole rule. Everything below is just spelling out what each half means.

## Hard on the code is allowed

Code, design choices, API decisions, PRs, commit messages, and architectural calls are all fair game for blunt criticism. You can say a piece of code is:

- bad, broken, crap, garbage, trash
- stupid, dumb, asinine, idiotic, half-baked
- incompetent, lazy, sloppy
- the worst design you have ever seen

If the criticism describes the work, it is in scope. The maintainer would rather get "this is garbage, here is why" than a paragraph of softening language wrapped around the same point. Direct review saves time. Time is the only thing this project genuinely tries to protect.

Strong opinions, clearly stated, with no hedging, are encouraged. "I think maybe perhaps this could possibly be improved" is not useful. "This is wrong, do X instead" is.

## Attacks on the person are not allowed

The bright line is whether the comment is about the work or about the human who produced it.

- "This code is garbage" describes the code. Allowed.
- "You are garbage" describes the person. Not allowed.
- "This is a stupid design" describes the design. Allowed.
- "You are stupid" describes the person. Not allowed.

Same word, different target, different rule. The test is: replace the subject of the sentence with the code under review. If the sentence still makes sense, it is about the code. If it does not, it is about the person.

Other things that are not allowed regardless of phrasing:

- Harassment: repeated unwanted contact, doxxing, sexual advances, threats. The usual.
- Sustained bad-faith argument: trolling, deliberate time-wasting, refusal to engage with technical responses.

## Enforcement

The maintainer decides. Reports go to `coroamaisaiah@gmail.com`. Escalation path is: warning, temporary block, permanent block. There is no committee, no appeals process, no public adjudication. If you think the maintainer got it wrong, fork the project. The license permits it.

## Note on platform rules

GitHub's site-wide Terms of Service and Community Guidelines apply on top of this document. GitHub can act on reports about content in this repository regardless of what is permitted here. Nothing in this CoC overrides that.

## Why this is short

Long codes of conduct invite legalistic argument about whether a specific phrase technically violates a specific clause. This one is short on purpose. The rule above is the rule. Apply it in good faith.
